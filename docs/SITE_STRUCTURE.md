# Структура сайта: страницы и блоки для вёрстки

Список страниц и блоков, чтобы верстать по шагам и не переделывать всё с нуля.

## Страницы

| № | Страница | Назначение |
|---|----------|------------|
| 1 | Главная (`index.html`) | Каталог с фильтрами (все товары, музыка, йога, для детей, дисциплина) и сеткой карточек. Отдельной страницы каталога нет. |
| 2 | (Позже) Карточка товара | Один плакат: фото, описание, кнопки Озон / WB / Цифровая версия |
| 3 | Контакты | Только в подвале (email, мессенджер), отдельной страницы нет |
| 4 | (Позже) Страница оплаты / успеха | После покупки цифровой версии |

Лендинги под рекламу (отдельные страницы под категории) можно добавить по тому же шаблону, не трогая уже готовые.

## Блоки для главной

- **Шапка:** название магазина «Komlew.posters» (текстом, без логотипа), навигация: Главная, Каталог (якорь #catalog). Контакты — в подвале.
- **Каталог (на главной):** под шапкой. Фильтры: Все товары, по инструментам (Гитара, Фортепиано, Бас-гитара, Укулеле, Барабаны), Йога, Для детей, Дисциплина. Сетка карточек (фото 3∶4, название, кнопки Ozon / WB / Цифровая версия). Данные из `site/data/products.json`.
- **Подвал:** контакты (email, Telegram), ссылки на Озон/WB, позже — политика конфиденциальности.

**Решения:** отдельной страницы каталога нет — все товары на главной. Контакты только в подвале. Бренд — Komlew.posters.

## Каталог (блок на главной)

- Фильтры: Все товары; по инструментам — Гитара, Фортепиано, Бас-гитара, Укулеле, Барабаны (сопоставление по id товара в `catalog.js`); по категориям — Йога, Для детей, Дисциплина. Музыка как одна категория убрана; «Программирование» и прочие — только в «Все товары».
- Сетка карточек: соотношение изображения 3∶4, `object-fit: cover`, название до 2 строк, кнопки Ozon / WB / Цифровая версия (если есть).
- Данные: `site/data/products.json`. При изменении контента в `content/products/` нужно перегенерировать (скрипт в корне или вручную).

## Блоки для карточки товара

- Шапка.
- **Хлебные крошки или кнопка «В каталог»** — обязательно. Из карточки пользователь должен попасть обратно в каталог без «назад в браузере». Пример: Главная → Каталог → Название плаката.
- Фото плаката, название, полное описание.
- Кнопки: «Купить на Озоне», «Купить на Wildberries», «Купить цифровую версию» (цена на сайте) — явно, не прятать в «Подробнее».
- Подвал.

Вёрстку удобно делать в порядке: главная → каталог → карточка товара. Каждый шаг — отдельный коммит.

---

## Требования из референсов (Озон/WB)

- **Карточка товара:** ссылка «В каталог» или хлебные крошки (подробнее — docs/REFERENCES_OZON_WB.md).
- **Каталог:** соотношение сторон карточек 3∶4; категории/фильтры видимые.
- **Единый вид:** одни и те же шапка, подвал, кнопки «Озон / WB / Цифра» на всех страницах.

## Что ещё продумать (до или во время вёрстки)

Чек-листы по каждой теме — в **docs/BACKLOG.md**. Кратко:

- **Доступность** — alt у картинок, контраст, фокус с клавиатуры.
- **Мета и SEO** — свой title и description у каждой страницы, у карточки — из данных товара.
- **Аналитика** — цели в Метрике: клики Озон / WB / Цифра, переход в каталог.
- **Загрузка и пустые состояния** — заглушка каталога, плейсхолдер картинок, «товар не найден».
- **Поиск** — категории на виду; поиск по названию — по желанию, позже.
- **Disclaimer про цены** — «Цена и наличие — на сайте маркетплейса».
- **Подписка** — форма email после витрины.
- **404** — своя страница со ссылками на главную и каталог.
- **Ссылки Ozon/WB** — периодическая проверка.

Эти пункты не блокируют старт вёрстки; порядок внедрения — в docs/BACKLOG.md.
