# Система файлов: порядок и правила

Одна функция — один файл. Один товар — одна папка (данные и картинки не пересекаются с другими товарами). Так проще не допускать ошибок и держать проект в порядке.

---

## 1. Товары (контент)

| Правило | Где |
|--------|-----|
| Один товар = одна папка | `content/products/<id>/` |
| ID товара = имя папки | `poster-1`, `azbuka`, `tabela` — без пробелов, латиница/цифры/дефис |
| Данные товара в одном файле | `content/products/<id>/product.json` |
| Список всех товаров (порядок для каталога) | `content/products/index.json` |

**Добавление нового товара:**
1. Создать папку `content/products/<новый-id>/`.
2. Положить туда `product.json` (см. пример в `content/products/example/`).
3. Добавить `<новый-id>` в `content/products/index.json`.

Картинки этого товара — только в `site/images/products/<тот-же-id>/` (см. ниже). Данные и картинки одного товара связаны одним ID, другие товары не трогаем.

---

## 2. Картинки

| Назначение | Путь | Правило |
|------------|------|--------|
| Картинки одного товара | `site/images/products/<id>/` | Одна папка на товар, ID совпадает с папкой в `content/products/` |
| Общие ресурсы (логотип, иконки, лого Озон/WB) | `site/images/shared/` | Всё, что не привязано к конкретному товару |

**Имена файлов внутри папки товара** — на ваше усмотрение, например:
- `main.jpg` — основное фото для карточки и страницы товара;
- `thumb.jpg` — превью для каталога (опционально).

В `product.json` указываете путь относительно сайта, например: `images/products/poster-1/main.jpg`. Так картинки разных товаров не пересекаются.

---

## 3. Стили (CSS)

| Правило | Где |
|--------|-----|
| Один блок/функция = один файл | `site/css/<имя-блока>.css` |
| Общие переменные | `site/css/tokens.css` |
| Сброс и базовая типографика | `site/css/base.css` |
| Шапка | `site/css/header.css` |
| Подвал | `site/css/footer.css` |
| Каталог | `site/css/catalog.css` |
| Карточка товара | `site/css/product.css` |
| Главная (герой, блок «Где купить» и т.д.) | `site/css/home.css` |

Не смешивать стили разных блоков в одном файле. Подключать в HTML только нужные для страницы.

---

## 4. Скрипты (JS)

| Правило | Где |
|--------|-----|
| Одна функция = один файл | `site/js/<имя-функции>.js` |
| Меню (мобильное открытие/закрытие) | `site/js/menu.js` |
| Каталог (подстановка карточек из данных, если нужно) | `site/js/catalog.js` |
| Страница товара (подстановка данных по ID, если нужно) | `site/js/product.js` |

Каждый файл отвечает за одну задачу. Не дублировать одну и ту же логику в нескольких файлах.

---

## 5. Страницы (HTML)

| Страница | Файл |
|----------|------|
| Главная | `site/index.html` |
| Каталог | `site/catalog.html` |
| Карточка товара | `site/product.html` (ID товара в URL: `?id=poster-1` или путь вида `product/poster-1.html` — решим при вёрстке) |

Общие блоки (шапка, подвал) можно подключать через include/шаблоны или копировать — в зависимости от того, будет ли сборка или чистый статический сайт.

---

## 6. Сводная схема

```
content/
  products/
    index.json              # список id товаров (порядок)
    <id>/
      product.json          # данные одного товара
    example/
      product.json

site/
  index.html
  catalog.html
  product.html
  css/
    tokens.css
    base.css
    header.css
    footer.css
    home.css
    catalog.css
    product.css
  js/
    menu.js
    catalog.js
    product.js
  images/
    shared/                 # логотипы, иконки
    products/
      <id>/                 # только картинки этого товара
        main.jpg
        thumb.jpg
```

---

## 7. Краткие правила «чтобы не ломать»

1. **ID товара** — один и тот же в `content/products/<id>/` и `site/images/products/<id>/`.
2. **Новый товар** — новая папка в `content/products/` и новая папка в `site/images/products/`; в `index.json` добавить id.
3. **Новый блок на странице** — новый CSS-файл (и при необходимости JS), не дописывать в общую «кучу».
4. **Общие картинки** — только в `site/images/shared/`, не в папках товаров.

Если держаться этих правил, пересечений и путаницы между товарами и блоками не будет.
