# Карточка товара и сетка каталога

Решения по карточке (MVP) и сохранённые параметры grid из ТЗ. Цены и отзывы в контенте добавим позже.

---

## 1. Сетка каталога (сохраняем из ТЗ)

| Параметр | Значение |
|----------|----------|
| Карточек в ряд (десктоп) | 5 |
| Ширина контейнера | 1280–1400px |
| Расстояние между карточками | 16–24px |
| Ширина карточки | 220–240px |
| Высота карточки | 360–400px |

Пример сетки:
```
[card][card][card][card][card]
[card][card][card][card][card]
```

---

## 2. Карточка MVP (главная и каталог)

**Пока делаем:**
- Изображение товара (3:4, `object-fit: cover`, скругление 12–16px).
- Название товара (макс. 2 строки, `text-overflow: ellipsis`).
- Рейтинг и количество отзывов (блок в вёрстке; данные в контенте добавим позже).

**Не делаем сейчас:**
- Цена, старая цена, скидка — добавим позже, когда будет источник данных.

**Один компонент карточки:** одна разметка/стили. Блоки «цена», «скидка», «рейтинг/отзывы» показываем только если в данных есть соответствующие поля — чтобы потом включить без переделки.

---

## 3. Структура карточки (MVP)

```
┌─────────────────────┐
│     IMAGE 3:4        │
│  (object-fit: cover) │
├─────────────────────┤
│ Название (2 строки)  │
│ ⭐ рейтинг • N отзывов │  ← данные позже
└─────────────────────┘
```

Плюс кнопки «Озон» / «WB» / «Цифровая версия» (на карточке в каталоге или на странице товара — по текущему плану).

---

## 4. Параметры из ТЗ (для вёрстки)

**Изображение:**
- Соотношение 3:4, ширина 100% карточки, высота ≈240–280px.
- `object-fit: cover`, `border-radius: 12–16px`.

**Название:**
- font-size: 14px, font-weight: 400–500, color: #333.
- Макс. 2 строки, `text-overflow: ellipsis`.

**Рейтинг и отзывы (когда будут данные):**
- Звезда: color orange, 12–14px.
- Текст: font-size 13px, color #666.
- Формат: «⭐ 4.9 • 94 отзыва».

**Отступы внутри карточки:**
- изображение → следующий блок: 8px;
- между блоками: 4–6px;
- внутренний padding карточки: 8–12px.

**Шрифт:** Inter (нейтральный UI).

---

## 5. Контент (поля добавим позже)

В `product.json` пока нет:
- `rating` (число, например 4.9),
- `reviews_count` (число),
- `price`, `price_old`, `discount_percent` (когда решим показывать цены).

Структуру карточки и опциональные блоки делаем так, чтобы при добавлении полей достаточно было заполнить данные — вёрстку не переделывать.
